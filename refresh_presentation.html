Step back

What is a refresh, really?

It asks every provider one question: “what does the world look like right now?”

Refresh
fresh view of the world vs State
what Terraform last knew = Drift

Refresh and drift detection are the same operation.

One knob, two systems

So there's really one question:
how often should we look at the world?

Every Run refreshes

A fresh view on every plan — even seconds apart. Maximum cost, maximum redundancy.

Drift runs every 24h

In HCP Terraform, drift detection looks once a day. The same operation — on a totally different clock.

Two systems answering the same question with wildly different — and disconnected — answers.

The insight

We keep re-discovering
the same fresh view.

A refresh is just data: a snapshot of the world at a moment in time.

What if the refresh were a reusable artifact — a cache that any plan could read?

The unifying solution

Two moves.

01

A refresh artifact

Terraform Core can output the refreshed snapshot, and a later plan can consume it instead of refreshing live. plan -refresh-out → plan -with-refresh

02

Drift every hour

Shorten drift detection from 24h → 1h. Configurable by org / TFE admins, per workspace, project, or stack. Each drift run refreshes the cache.

One produces the cache. The other keeps it fresh. Same operation, finally shared.

How it ties together

The refresh moves off the critical path.

🕐

Hourly drift

A background drift run refreshes the world on a schedule.

📦

Writes the artifact

That refresh is saved as a reusable cache.

⚡

Runs reuse it

Every plan / apply reads the cache — no live refresh.

🚀

Runs are fast

The bottleneck is gone. Plans are seconds, not minutes.

Refresh once, in the background. Reuse it everywhere — humans, CI, and agents.

Open questions

Not every workspace needs the same clock.

  • How often should each workspace refresh? By usage frequency? By customer tier?
  • Admin-configurable per workspace / project / stack — sensible defaults, escape hatches.
  • Smarter scheduling: refresh hot workspaces often, cold ones rarely.
